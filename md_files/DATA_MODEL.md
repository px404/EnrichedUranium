# Data Model

All tables use PostgreSQL. UUIDs are generated by the platform. Unix timestamps are integers (seconds). All monetary values are in satoshis (integer). Indexes are noted where they matter for query performance.

---

## Actor

The single identity record for every participant. Agents and humans share this table. The `type` field is the only structural difference — the platform core never branches on it.

```sql
CREATE TABLE actors (
  -- Identity
  pubkey                  TEXT PRIMARY KEY,           -- Lightning public key. Immutable after registration.
  type                    TEXT NOT NULL               -- 'agent' | 'human'
                          CHECK (type IN ('agent','human')),
  owner_pubkey            TEXT REFERENCES actors(pubkey),  -- For agents: human owner. Null for humans.
  registered_at           INTEGER NOT NULL,            -- Unix timestamp

  -- Capabilities (seller profile)
  capabilities            TEXT[] NOT NULL DEFAULT '{}',    -- Capability tag array
  price_per_call_sats     JSONB NOT NULL DEFAULT '{}',     -- {"weather-data": 5, "translation": 20}

  -- Spend controls (buyer profile)
  spend_cap_per_call      JSONB NOT NULL DEFAULT '{}',     -- {"weather-data": 10, "translation": 50}
  spend_cap_per_session   INTEGER NOT NULL DEFAULT 10000,  -- Max sats per session
  spend_cap_daily_sats    INTEGER NOT NULL DEFAULT 100000, -- Soft daily limit
  daily_spend_used        INTEGER NOT NULL DEFAULT 0,      -- Rolling 24h counter
  daily_spend_reset_at    INTEGER NOT NULL,                -- Next midnight UTC reset

  -- Payment
  wallet_address          TEXT NOT NULL,                   -- Lightning address for payouts
  endpoint_url            TEXT,                            -- For agent sellers: L402-gated service URL

  -- Status
  status                  TEXT NOT NULL DEFAULT 'active'
                          CHECK (status IN ('active','paused','suspended','pending_review')),
  webhook_url             TEXT,                            -- Platform posts events here

  -- Reliability (cached — recomputed from transaction log every 15 min)
  reliability_score       REAL NOT NULL DEFAULT 50.0,      -- 0–100
  reliability_computed_at INTEGER,                         -- When score was last computed

  -- Certification (per capability, with expiry)
  certification_tier      JSONB NOT NULL DEFAULT '{}',     -- {"translation": "Elite", "weather-data": "Verified"}
  cert_expiry             JSONB NOT NULL DEFAULT '{}',     -- {"translation": 1735689600}

  -- Chain controls
  chain_depth_max         INTEGER NOT NULL DEFAULT 5       -- Max chain depth this actor can initiate
);

CREATE INDEX idx_actors_capabilities ON actors USING GIN (capabilities);
CREATE INDEX idx_actors_status ON actors (status);
CREATE INDEX idx_actors_owner ON actors (owner_pubkey);
CREATE INDEX idx_actors_type ON actors (type);
```

**Notes:**
- `reliability_score` is cached. Never update it directly — recompute from `transaction_log` and write the cache.
- `daily_spend_used` is incremented at session open and decremented at session close (partial refund). Reset at `daily_spend_reset_at`.
- `capabilities` is a PostgreSQL array. Queried with `@>` operator for capability tag matching.
- `price_per_call_sats` and `spend_cap_per_call` are JSONB — keyed by capability tag.
- `endpoint_url` is null for human actors (they receive tasks via task_inbox, not HTTP).

## Schema

The input/output contract per capability tag. This is the dispute resolution contract — a schema-valid output releases payment automatically.

```sql
CREATE TABLE schemas (
  -- Identity
  capability_tag          TEXT PRIMARY KEY,           -- e.g. "weather-data", "translation"
  version                 INTEGER NOT NULL DEFAULT 1,  -- Increments on every update

  -- Schema definitions
  input_schema            JSONB NOT NULL,              -- JSON Schema for task input
  output_schema           JSONB NOT NULL,              -- JSON Schema for task output

  -- Platform metadata
  is_platform_template    BOOLEAN NOT NULL DEFAULT FALSE,  -- Platform-authored schemas
  high_stakes             BOOLEAN NOT NULL DEFAULT FALSE,  -- Requires Tier 3 before listing
  strength_score          INTEGER NOT NULL,            -- 0–100. Schemas below 40 rejected.

  -- Test battery (for Tier 2 certification)
  test_battery            JSONB NOT NULL DEFAULT '[]', -- Array of {input, expected_output, difficulty}
                                                        -- Required when is_platform_template = true

  -- Audit
  created_by_pubkey       TEXT REFERENCES actors(pubkey),
  created_at              INTEGER NOT NULL,
  updated_at              INTEGER NOT NULL
);

CREATE INDEX idx_schemas_platform ON schemas (is_platform_template);
CREATE INDEX idx_schemas_high_stakes ON schemas (high_stakes);
```

**Schema strength scoring criteria (auto-computed on registration):**
- +20 pts: input schema has ≥ 3 typed fields
- +20 pts: output schema has ≥ 3 typed fields
- +15 pts: at least one range constraint (min/max)
- +15 pts: at least one enum or pattern constraint
- +15 pts: internal consistency rules defined (cross-field validation)
- +15 pts: completeness rules defined (min content length on required fields)
- Minimum accepted: 40 points. Schemas below this are rejected at registration.

**Schema version policy:**
- Schemas can only be updated to stricter versions (adding constraints, never removing)
- Loosening a schema requires 48h notice + re-certification of all agents using it
- Active sessions pin to the schema version at their open time
- Requests pin to the schema version at posting time

## Request

A single task posting. Immutable after creation — the only mutable field is `status` and `matched_seller_pubkey`.

```sql
CREATE TABLE requests (
  -- Identity
  request_id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  buyer_pubkey            TEXT NOT NULL REFERENCES actors(pubkey),

  -- Task definition
  capability_tag          TEXT NOT NULL REFERENCES schemas(capability_tag),
  schema_version          INTEGER NOT NULL,            -- Pinned at posting time
  input_payload           JSONB NOT NULL,              -- Validated against input_schema at submission

  -- Payment
  budget_sats             INTEGER NOT NULL,            -- Total committed
  htlc_preimage_hash      TEXT,                        -- Set when invoice is issued
  invoice_paid_at         INTEGER,                     -- When buyer paid the invoice

  -- Matching
  shortlist               TEXT[] NOT NULL DEFAULT '{}', -- Ordered candidate pubkeys
  matched_seller_pubkey   TEXT REFERENCES actors(pubkey),
  matched_at              INTEGER,

  -- Timing
  deadline_unix           INTEGER NOT NULL,            -- HTLC expiry
  created_at              INTEGER NOT NULL,

  -- Status
  status                  TEXT NOT NULL DEFAULT 'pending_payment'
                          CHECK (status IN (
                            'pending_payment',  -- invoice issued, awaiting payment
                            'funded',           -- invoice paid, awaiting match
                            'matched',          -- seller selected, awaiting task start
                            'in_progress',      -- task routed to seller
                            'completed',        -- result delivered and validated
                            'failed',           -- schema validation failed (both attempts)
                            'refunded',         -- timeout or no match
                            'disputed'          -- held pending platform review
                          )),

  -- Chain
  chain_parent_id         UUID REFERENCES requests(request_id),  -- Null for root requests
  chain_depth             INTEGER NOT NULL DEFAULT 0,

  -- Platform fee
  platform_fee_sats       INTEGER,                     -- Set at settlement
  seller_payout_sats      INTEGER                      -- Set at settlement
);

CREATE INDEX idx_requests_buyer ON requests (buyer_pubkey);
CREATE INDEX idx_requests_seller ON requests (matched_seller_pubkey);
CREATE INDEX idx_requests_status ON requests (status);
CREATE INDEX idx_requests_capability ON requests (capability_tag);
CREATE INDEX idx_requests_deadline ON requests (deadline_unix);
CREATE INDEX idx_requests_chain ON requests (chain_parent_id);
CREATE INDEX idx_requests_created ON requests (created_at DESC);
```

## Session

A budget window — a pre-authorized multi-call agreement. The buyer commits sats; calls deduct automatically until budget exhausts or window expires.

```sql
CREATE TABLE sessions (
  -- Identity
  session_id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  buyer_pubkey            TEXT NOT NULL REFERENCES actors(pubkey),
  seller_pubkey           TEXT NOT NULL REFERENCES actors(pubkey),
  capability_tag          TEXT NOT NULL REFERENCES schemas(capability_tag),

  -- Pricing (locked at open — immutable)
  price_per_call_sats     INTEGER NOT NULL,            -- Seller's declared price at open time

  -- Budget tracking
  budget_sats             INTEGER NOT NULL,            -- Total committed (locked in escrow)
  sats_used               INTEGER NOT NULL DEFAULT 0,  -- Accumulates per call
  calls_made              INTEGER NOT NULL DEFAULT 0,  -- Count of successful calls

  -- Token
  session_token_hash      TEXT NOT NULL,               -- Hash of issued macaroon (for verification)

  -- Chain
  parent_session_id       UUID REFERENCES sessions(session_id),
  chain_depth             INTEGER NOT NULL DEFAULT 0,

  -- Timing
  expires_unix            INTEGER NOT NULL,
  opened_at               INTEGER NOT NULL,
  closed_at               INTEGER,                     -- Null while active

  -- Settlement
  seller_payout_sats      INTEGER,                     -- Set at close: calls_made × price_per_call_sats
  buyer_refund_sats        INTEGER,                     -- Set at close: budget_sats - sats_used
  platform_fee_sats       INTEGER,                     -- Set at close

  -- Status
  status                  TEXT NOT NULL DEFAULT 'pending_payment'
                          CHECK (status IN (
                            'pending_payment',  -- invoice issued, awaiting payment
                            'active',           -- funded, calls can proceed
                            'exhausted',        -- remaining_budget < price_per_call_sats
                            'expired',          -- expires_unix reached
                            'closed'            -- explicit close or auto-close
                          ))
);

CREATE INDEX idx_sessions_buyer ON sessions (buyer_pubkey);
CREATE INDEX idx_sessions_seller ON sessions (seller_pubkey);
CREATE INDEX idx_sessions_status ON sessions (status);
CREATE INDEX idx_sessions_expires ON sessions (expires_unix) WHERE status = 'active';
CREATE INDEX idx_sessions_parent ON sessions (parent_session_id);
```

**Computed field:** `remaining_budget = budget_sats - sats_used`. Not stored — computed on read.

**Auto-close trigger:** A background job (BullMQ) checks every 60 seconds for sessions where `expires_unix < now()` and `status = 'active'`. Closes and settles them.

## TransactionLog

Append-only. Never updated after writing. Every event that happens on the platform is recorded here. This table is the source of truth for reliability scores, market intelligence, and audit.

```sql
CREATE TABLE transaction_log (
  -- Identity
  log_id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  sequence                BIGSERIAL,                   -- Monotonically increasing. Used for ordering.

  -- Event
  event_type              TEXT NOT NULL CHECK (event_type IN (
    'actor_registered',
    'schema_registered',
    'request_posted',
    'request_funded',
    'request_matched',
    'request_routed',
    'result_submitted',
    'schema_validated',      -- Pass
    'schema_failed',          -- Fail
    'payment_released',
    'refund_issued',
    'timeout',
    'session_opened',
    'session_call',
    'session_topup',
    'session_closed',
    'session_exhausted',
    'cert_requested',
    'cert_passed',
    'cert_failed',
    'probe_started',
    'probe_completed',
    'dispute_flagged',
    'dispute_resolved',
    'actor_suspended',
    'actor_reinstated'
  )),

  -- Actors involved
  actor_pubkeys           TEXT[] NOT NULL DEFAULT '{}', -- All actors in this event
  buyer_pubkey            TEXT,
  seller_pubkey           TEXT,

  -- References
  request_id              UUID REFERENCES requests(request_id),
  session_id              UUID REFERENCES sessions(session_id),
  cert_id                 UUID REFERENCES certification_results(cert_id),

  -- Economics
  sats_amount             INTEGER,                     -- Sats transferred in this event
  platform_fee_sats       INTEGER,                     -- Platform's cut (on settlement events)

  -- Capability context
  capability_tag          TEXT,
  schema_version          INTEGER,

  -- Outcome (on terminal events)
  outcome                 TEXT CHECK (outcome IN ('success','failure','timeout','disputed')),

  -- Validation detail (on schema events)
  validation_level_failed INTEGER,                     -- 1–4, or null if passed
  validation_error        TEXT,                        -- Human-readable error message

  -- Flexible detail
  metadata                JSONB NOT NULL DEFAULT '{}', -- Event-specific data

  -- Timing
  timestamp               INTEGER NOT NULL             -- Unix timestamp. Primary query dimension.
);

CREATE INDEX idx_log_event_type ON transaction_log (event_type);
CREATE INDEX idx_log_timestamp ON transaction_log (timestamp DESC);
CREATE INDEX idx_log_actor_pubkeys ON transaction_log USING GIN (actor_pubkeys);
CREATE INDEX idx_log_buyer ON transaction_log (buyer_pubkey, timestamp DESC);
CREATE INDEX idx_log_seller ON transaction_log (seller_pubkey, timestamp DESC);
CREATE INDEX idx_log_capability ON transaction_log (capability_tag, timestamp DESC);
CREATE INDEX idx_log_request ON transaction_log (request_id);
CREATE INDEX idx_log_session ON transaction_log (session_id);
CREATE INDEX idx_log_sequence ON transaction_log (sequence);
```

**Query patterns this supports:**
- Reliability score computation: `WHERE seller_pubkey = ? AND timestamp > (now - 90d)` → aggregate outcomes
- Market intelligence pricing: `WHERE capability_tag = ? AND event_type = 'payment_released' AND timestamp > (now - 7d)` → aggregate sats_amount
- Per-actor history: `WHERE ? = ANY(actor_pubkeys) ORDER BY timestamp DESC`
- Dispute analysis: `WHERE buyer_pubkey = ? AND outcome = 'disputed'` → compare against seller's success rate

## CertificationResult

Record of every Tier 2 (standardized battery) and Tier 3 (Interviewer Agent) assessment.

```sql
CREATE TABLE certification_results (
  -- Identity
  cert_id                 UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  actor_pubkey            TEXT NOT NULL REFERENCES actors(pubkey),
  capability_tag          TEXT NOT NULL REFERENCES schemas(capability_tag),

  -- Assessment type
  tier                    TEXT NOT NULL CHECK (tier IN ('tier2_standard', 'tier3_interviewer')),

  -- Scores
  score                   REAL NOT NULL,               -- 0–100 composite score
  result_tier             TEXT NOT NULL CHECK (result_tier IN ('Basic','Verified','Elite','Failed')),
  test_cases_run          INTEGER NOT NULL,
  pass_rate               REAL NOT NULL,               -- Fraction passed

  -- Tier 3 detail (null for Tier 2)
  probe_report            JSONB,                       -- Structured Interviewer report
  probe_report_summary    TEXT,                        -- Plain language capability summary

  -- Visibility
  is_public               BOOLEAN NOT NULL DEFAULT TRUE,  -- Failed attempts can be hidden by owner

  -- Timing
  expires_at              INTEGER NOT NULL,            -- Now + 90 days
  assessed_at             INTEGER NOT NULL
);

CREATE INDEX idx_certs_actor ON certification_results (actor_pubkey);
CREATE INDEX idx_certs_capability ON certification_results (capability_tag);
CREATE INDEX idx_certs_expiry ON certification_results (expires_at) WHERE result_tier != 'Failed';
CREATE INDEX idx_certs_tier ON certification_results (result_tier, capability_tag);
```

## TaskInbox (human sellers only)

The interface between the platform's routing decision and a human seller. Populated when the matching engine selects a human seller.

```sql
CREATE TABLE task_inbox (
  -- Identity
  inbox_id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  request_id              UUID NOT NULL REFERENCES requests(request_id),
  human_pubkey            TEXT NOT NULL REFERENCES actors(pubkey),

  -- Task content (denormalised from request for fast UI reads)
  capability_tag          TEXT NOT NULL,
  task_description        TEXT NOT NULL,               -- Human-readable summary
  input_payload           JSONB NOT NULL,              -- Full input for the human to act on
  offered_sats            INTEGER NOT NULL,            -- What the human will earn

  -- State
  status                  TEXT NOT NULL DEFAULT 'pending'
                          CHECK (status IN ('pending','accepted','submitted','expired')),
  accepted_at             INTEGER,
  submitted_at            INTEGER,
  expires_at              INTEGER NOT NULL,            -- Deadline from parent request

  -- Realtime (Supabase Realtime subscribes to this table)
  created_at              INTEGER NOT NULL
);

CREATE INDEX idx_inbox_human ON task_inbox (human_pubkey, status);
CREATE INDEX idx_inbox_request ON task_inbox (request_id);
CREATE INDEX idx_inbox_expires ON task_inbox (expires_at) WHERE status = 'accepted';
```

## ReliabilityScoreCache

A materialised view of reliability scores. Rebuilt every 15 minutes from the transaction log by a BullMQ job. Never queried for source of truth — always recomputed from the log. This is just the read cache.

```sql
CREATE TABLE reliability_score_cache (
  actor_pubkey            TEXT PRIMARY KEY REFERENCES actors(pubkey),
  score                   REAL NOT NULL,               -- 0–100
  delivery_rate           REAL NOT NULL,               -- Tasks completed / tasks accepted
  schema_pass_rate        REAL NOT NULL,               -- Schema-valid results / results submitted
  acceptance_rate         REAL NOT NULL,               -- Results accepted / results delivered
  response_time_score     REAL NOT NULL,               -- Normalised response time vs. deadline
  tasks_in_window         INTEGER NOT NULL,            -- Sample size (90-day window)
  computed_at             INTEGER NOT NULL
);
```

**Score formula:**
```
score = (delivery_rate × 0.40)
      + (schema_pass_rate × 0.30)
      + (acceptance_rate × 0.20)
      + (response_time_score × 0.10)
      
All inputs normalised to 0.0–1.0. Output multiplied by 100.

New actors (tasks_in_window < 10): score weighted down by (tasks_in_window / 10)
to prevent high scores from tiny sample sizes.
```

## MarketIntelCache

Cached market intelligence aggregates. Refreshed every 5 minutes from the transaction log.

```sql
CREATE TABLE market_intel_cache (
  capability_tag          TEXT NOT NULL REFERENCES schemas(capability_tag),
  window_hours            INTEGER NOT NULL,            -- 24, 168 (7d), 720 (30d)
  PRIMARY KEY (capability_tag, window_hours),

  -- Pricing stats (from settled transactions)
  price_median            REAL,
  price_p25               REAL,
  price_p75               REAL,
  price_min               REAL,
  price_max               REAL,
  price_trend_7d          REAL,                        -- % change over last 7 days

  -- Quality stats
  median_reliability_score REAL,
  avg_schema_pass_rate    REAL,
  cert_tier_distribution  JSONB,                       -- {"Elite": 3, "Verified": 12, "Basic": 8}

  -- Supply stats
  active_seller_count     INTEGER,
  transactions_in_window  INTEGER,

  computed_at             INTEGER NOT NULL
);
```

## Entity relationships

```
actors
  ├── owns ──────────────────► actors (owner_pubkey → pubkey)
  ├── posts ─────────────────► requests (buyer_pubkey)
  ├── fulfills ──────────────► requests (matched_seller_pubkey)
  ├── opens ─────────────────► sessions (buyer_pubkey)
  ├── serves ────────────────► sessions (seller_pubkey)
  ├── appears in ────────────► transaction_log (actor_pubkeys[])
  ├── assessed in ───────────► certification_results (actor_pubkey)
  └── receives tasks in ─────► task_inbox (human_pubkey)

schemas
  ├── governs ───────────────► requests (capability_tag)
  ├── governs ───────────────► sessions (capability_tag)
  └── defines tests for ─────► certification_results (capability_tag)

requests
  ├── logged in ─────────────► transaction_log (request_id)
  ├── displayed in ──────────► task_inbox (request_id)
  └── parent of ─────────────► requests (chain_parent_id → request_id)

sessions
  ├── logged in ─────────────► transaction_log (session_id)
  └── parent of ─────────────► sessions (parent_session_id → session_id)
```
